# 实时做 T 工具研究设计

本文设计一个“开盘实时拉取指定股票数据、本地落盘、盘中分析、盘后复盘”的日内做 T 辅助工具。目标不是自动交易，而是为有底仓的 A 股持仓提供高抛低吸的量化参考。

## 核心判断

可行，但第一版必须克制：先做实时采集和解释型信号，不做自动下单。

原因：

- 百度财经 WebSocket 已验证能订阅 A 股实时快照，适合做盘中采集。
- `easy_tdx` 可作为主通达信源，补 K 线、指标、板块和资金；`mootdx` 只做 fallback。
- 同花顺问财和 `a-stock-data` 可补充盘前选股、热点、研报、龙虎榜等上下文。
- A 股普通账户是 T+1，做 T 必须基于已有底仓，工具应明确“无底仓不触发做 T 建议”。

## 产品目标

### MVP

输入：

- 股票列表，例如 `002463,600941,603986`。
- 可选底仓信息，例如持仓数量、成本价、计划做 T 数量。
- 可选交易风格，例如保守、均衡、激进。

输出：

- 实时 JSONL 数据文件。
- 每分钟分时分析摘要。
- 做 T 信号：高抛、低吸、回补、观望、禁止交易。
- 信号理由：价格位置、VWAP 偏离、量能变化、分时结构、板块/大盘背景。
- 盘后复盘：当天所有触发信号、假设成交、收益/风险评估。

非目标：

- 不做自动下单。
- 不承诺胜率。
- 不用未经验证的 X/Twitter 内容做事实依据。
- 不在第一版做全市场扫描。

## 数据源分工

| 数据源 | 用途 | 优先级 | 说明 |
|---|---|---|---|
| 百度财经 WebSocket | 盘中实时快照 | 主实时源 | 已验证 `snapshot` 订阅可用 |
| `easy_tdx` | K 线、指标、板块、资金、异动、集合竞价 | 主通达信源 | 新功能优先使用 |
| `mootdx` | K 线/行情 fallback | 备用 | 仅在 `easy_tdx` 不可用时降级 |
| 同花顺问财 | 盘前/盘后候选股筛选 | 选股源 | 需本地二次计算确认 |
| `a-stock-data` | 研报、新闻、热点、龙虎榜、资金 | 补充源 | 东财接口需限流 |
| X 本地缓存 | 市场关注线索 | 辅助 | 只读本地，未验证 |

## 盘中指标

| 指标 | 计算 | 用途 |
|---|---|---|
| VWAP | 累计成交额 / 累计成交量 | 判断均价线支撑/压力 |
| VWAP 偏离率 | `(price - vwap) / vwap * 100` | 高抛/低吸位置判断 |
| 分时高低点 | 当日实时 high/low | 判断突破、回撤、双底 |
| 1/3/5 分钟成交额变化 | 分钟增量滚动比较 | 判断放量/缩量 |
| 回撤幅度 | `(price - high_so_far) / high_so_far` | 判断冲高回落 |
| 修复幅度 | `(price - low_so_far) / low_so_far` | 判断急杀修复 |
| 相对开盘涨跌 | `(price - open) / open` | 判断开盘后强弱 |
| 相对昨收涨跌 | `(price - pre_close) / pre_close` | 判断全天强弱 |
| 分时结构 | 前高、前低、二次高低点 | 判断高抛/低吸形态 |

## 信号体系

### 信号枚举

| signal | 含义 | 动作 |
|---|---|---|
| `high_sell` | 高抛信号 | 减少部分底仓 |
| `low_buy` | 低吸信号 | 买回计划 T 仓 |
| `cover_back` | 回补信号 | 对已高抛仓位进行回补 |
| `hold` | 持有 | 不操作 |
| `watch` | 观察 | 接近条件但未触发 |
| `forbidden` | 禁止交易 | 风险区，不建议操作 |

### 禁止交易优先级

任何策略触发前先判断禁止条件：

- 无底仓，却触发做 T 建议。
- 开盘后前 3-5 分钟。
- 接近涨停/跌停。
- 当日数据缺失或 WebSocket 中断超过阈值。
- 大盘/板块急跌且个股没有独立强度。
- 成交量极低，价格跳动失真。

## 候选策略

### 策略 1：高开冲高回落高抛

适用：已有底仓，早盘强冲但承接转弱。

触发条件建议：

- `price_vs_pre_close_pct > 2%` 或开盘高开。
- 10-30 分钟内出现日内高点。
- 二次冲高不破前高，或突破后快速回落。
- 价格跌回 VWAP 附近或跌破 VWAP。
- 第二次冲高成交额低于第一次。

动作：高抛部分底仓。

失效条件：放量突破前高并站稳。

### 策略 2：低开急杀修复低吸

适用：趋势股低开急杀后出现承接。

触发条件建议：

- `price_vs_pre_close_pct < -2%`。
- 形成第一次低点后反弹。
- 二次下探不破前低，且成交额缩小。
- 重新站上短周期均价或 VWAP。

动作：低吸 T 仓。

失效条件：跌破二次低点或大盘/板块同步走弱。

### 策略 3：VWAP 回踩不破低吸

适用：强势股盘中回踩。

触发条件建议：

- 当日大部分时间价格在 VWAP 上方。
- 回踩 VWAP 附近，偏离率在 `-0.3% ~ 0.5%`。
- 回踩时成交额缩小。
- 重新放量上穿最近 3-5 分钟高点。

动作：低吸或持有。

失效条件：放量跌破 VWAP 且无法快速收回。

### 策略 4：冲高不过前高高抛

适用：震荡股或高位趋势股。

触发条件建议：

- 当日已有一次明显冲高。
- 第二次冲高接近前高但未突破。
- 第二次冲高量能弱于第一次。
- 价格回落跌破最近 3 分钟低点。

动作：高抛部分底仓。

失效条件：放量突破前高。

### 策略 5：板块共振低吸

适用：主线板块内核心股回踩。

触发条件建议：

- `easy_tdx board-ranking` 显示所属板块涨幅/成交额靠前。
- 个股回踩 VWAP 或 5/10 日线附近。
- 板块内涨多跌少。
- 个股相对板块不明显掉队。

动作：低吸或继续持有。

失效条件：板块强度下降或核心股集体转弱。

## 评分模型

每个信号满分 100，先用于排序和解释，不用于自动交易。

| 维度 | 分值 | 示例 |
|---|---:|---|
| 价格位置 | 25 | VWAP 偏离、前高/前低、开盘/昨收位置 |
| 量能结构 | 25 | 冲高缩量、下杀缩量、回升放量 |
| 分时形态 | 20 | 双底、冲高不过、VWAP 回踩 |
| 市场环境 | 15 | 大盘、板块、同题材强弱 |
| 个股上下文 | 15 | 盘前候选质量、研报/热点/龙虎榜/X线索 |

信号强度分级：

| 分数 | 等级 | 建议 |
|---:|---|---|
| `>= 80` | 强 | 可重点关注，但仍需人工确认 |
| `65-79` | 中 | 可观察或小仓位尝试 |
| `50-64` | 弱 | 只提示，不建议主动操作 |
| `< 50` | 无效 | 不输出交易建议 |

## 盘前准备

盘前应先生成候选池，不建议开盘临时全市场扫：

1. 同花顺问财筛缩量回踩均线。
2. `easy_tdx` 补趋势指标，例如 `ZHUOYAO`、`BIAS_SIGNAL`。
3. `a-stock-data` 补研报、热点、龙虎榜、新闻。
4. 读取 X 本地缓存作为关注度线索。
5. 最终输出 5-20 只候选股票，供开盘订阅。

候选池字段建议：

```json
{
  "code": "002463",
  "name": "沪电股份",
  "reason": "缩量回踩10日线，PCB/AI算力链，研报和X缓存提及较多",
  "watch_levels": {
    "support": [130.66, 131.47],
    "pressure": [140.82, 141.28]
  },
  "position": {
    "has_base_position": true,
    "base_shares": 1000,
    "planned_t_shares": 300
  }
}
```

## 盘后复盘与回测

### 盘后复盘

每日收盘后生成：

- 每只股票的分时曲线统计。
- 所有信号触发点。
- 假设按信号价成交的收益。
- 若信号失败，记录失败类型。
- 当日策略胜率和平均收益。

### 回测口径

做 T 回测不能只看买入后涨跌，要模拟底仓：

- 高抛：卖出底仓的一部分，之后低位回补。
- 低吸：买入 T 仓，之后高位卖出等量底仓。
- 若当天没有反向机会，则计为未完成 T，按收盘风险评估。

### 指标

| 指标 | 含义 |
|---|---|
| 信号次数 | 当天触发次数 |
| 完成 T 次数 | 有买有卖形成闭环 |
| 胜率 | 完成 T 后收益为正比例 |
| 平均 T 收益 | 每次 T 的价差收益 |
| 最大逆向浮亏 | 信号触发后最差浮亏 |
| 信号后 5/15/30 分钟表现 | 判断信号时效 |
| 未完成 T 风险 | 当天只买未卖或只卖未买的风险 |

该功能作为独立工具放在 `apps/intraday-t/`，不放入 `apps/TradingAgents-astock/`，也不直接散放在仓库顶层。建议目录：

```text

    ├── pyproject.toml
    ├── scripts/
```

### `intraday_collector.py`

职责：

- 读取股票列表。
- 建立百度财经 WebSocket 连接。
- 订阅 `snapshot`。
- 标准化字段。
- 落盘 raw JSONL。
- 断线重连。

### `intraday_aggregator.py`

职责：

- 读取 raw JSONL。
- 生成 1 分钟聚合。
- 计算 VWAP、成交额增量、分时高低点。
- 输出 minute JSONL。

### `intraday_signal_engine.py`

职责：

- 读取 minute 数据和候选池上下文。
- 判断禁止交易区。
- 执行策略规则。
- 输出信号 JSONL。

### `intraday_report.py`

职责：

- 盘中打印简报。
- 盘后生成 Markdown/HTML 复盘报告。

### `tdx_provider.py`

职责：

- 主用 `easy_tdx`。
- fallback 到 `mootdx`。
- 提供统一 K 线/指标/板块/资金接口。

## 实现阶段

### 阶段 1：最小采集器

验收：

- 能订阅 1-3 只股票。
- 能连续落盘 30 分钟。
- JSONL 字段稳定。
- Ctrl+C 后文件完整。

### 阶段 2：分钟聚合

验收：

- 能从 raw 生成 1m 数据。
- 成交额/成交量增量非负。
- VWAP 与百度返回均价接近。

### 阶段 3：基础信号

验收：

- 输出 `high_sell`、`low_buy`、`hold`、`forbidden`。
- 每个信号包含理由。
- 开盘前 5 分钟不会输出交易建议。

### 阶段 4：盘后复盘

验收：

- 能列出当天所有信号。
- 能计算假设 T 收益。
- 能标记失败信号。

### 阶段 5：焚诀沉淀

验收：

- 至少跑满 5-10 个交易日样本。
- 每个候选策略有胜率、平均收益、最大逆向浮亏。
- 只把表现稳定的策略写成“焚诀”。

## 风险与边界

- 数据源中断会导致信号失真，必须有断线标记。
- WebSocket 快照不是逐笔成交，不能当 tick 级回测。
- 做 T 依赖底仓和交易纪律，工具只能辅助判断。
- 盘中信号容易受大盘和板块影响，不能只看单股。
- 第一版不接券商下单接口，避免自动交易风险。
